# Evaluating Machine Translation

A comprehensive pipeline for evaluating the quality of Machine Translation (MT) models. This repository implements and compares various evaluation metrics to assess translation performance against reference datasets.

## 📌 Overview

Evaluating machine translation is a complex task because a single source sentence can have multiple valid translations. This project provides a structured framework to evaluate candidate translations generated by MT models against ground-truth references using both automated lexical metrics and modern embedding-based metrics.

## 🚀 Features

*   **Lexical Metrics:** Implementation/integration of standard string-matching metrics:
    *   **BLEU** (Bilingual Evaluation Understudy) — measures n-gram precision.
    *   **TER** (translation edit rate) — measures n-gram recall (useful if looking at ROUGE-L).
    *   **COMET** (Crosslingual Optimized Metric for Evaluation of Translation) — includes stemming and synonym matching.
*   **Semantic/Embedding Metrics:** (If applicable) Integration of modern trained metrics like **BERTScore** or **COMET** to capture semantic similarity beyond exact word matches.
*   **Preprocessing Pipeline:** Text normalization, tokenization, and handling of language-specific punctuation quirks.
*   **Analysis Tools:** Scripts to calculate corpus-level scores, sentence-level scores, and generate error distribution reports.
